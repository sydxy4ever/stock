# analyze_turnover_surge.py 功能总结

该脚本是一个用于股票换手率异动分析的工具，旨在识别在特定行业背景下出现显著换手率激增的龙头股，并跟踪其后续的市场表现。

## 1. 核心目标
以某个交易日（Day1）为基准，在 **申万2021三级行业** 的龙头股（市值 Top-N）中，筛选出近期（5日）平均换手率相对于基线（30日）出现显著放大（默认 1.8 倍以上）且 Day1 涨幅超过 3% 的股票，并记录其后 10 个交易日的详细指标。

## 2. 分析逻辑与时间窗口

分析过程涉及三个核心时间段（均为交易日）：

*   **基准期 (Baseline Period):** `Day(-35)` ~ `Day(-6)`，共 30 个交易日。计算该期间的平均换手率作为参考基准。
*   **观察期 (Observation Period):** `Day(-5)` ~ `Day(-1)`，共 5 个交易日。计算此期间的平均换手率并与基准期对比。
*   **跟踪期 (Tracking Period):** `Day(1)` ~ `Day(10)`，共 10 个交易日。记录触发后的每日股价、涨跌幅、换手率及均线状态。

## 3. 筛选策略 (Strategy)

*   **选股范围:** 申万 2021 三级行业。
*   **龙头定义:** 每个三级行业中按市值排序的前 $N$ 名（默认为 Top 3）。
*   **触发条件:**
    1.  **换手率倍率:** 观察期平均换手率 $\ge$ 基线期平均换手率的 1.8 倍。
    2.  **Day1 表现:** Day1 当天的股价涨幅必须 $\ge$ 3%。
    3.  **有效数据:** 基线期内至少有 15 个有效交易日数据。

## 4. 输出指标汇总

脚本生成的 CSV 或数据库记录包含以下关键字段：

*   **基础信息:** 行业代码/名称、股票代码/名称、市值排名。
*   **触发指标:** 基准换手率、近期换手率、触发比例 (Trigger Ratio)。
*   **Day1 快照:** Day1 涨幅、收盘价、均线位置（MA5/20/60/200）以及收盘价与各均线的比例关系。
*   **跟踪数据:** Day1 后的第 $n$ 天（Offset）、日期、收盘价、换手率、换手率放大倍数、当日涨幅、均线状态等。

## 5. 主要功能模块

*   **交易日历加载:** 从数据库中自动提取或生成准确的交易日序列。
*   **三级行业判断:** 通过行业代码规则精确识别申万三级行业。
*   **自动化批处理:** 支持分析单个日期或指定日期范围内的所有交易日。
*   **持久化存储:** 
    *   **SQLite:** 结果自动存入 `turnover_surge.db`，支持断点续跑（跳过已处理日期）。
    *   **CSV:** 每个分析日生成一个独立的 CSV 文件。

## 6. 使用方法

```bash
# 1. 分析单个交易日
python analyze_turnover_surge.py --day1 2026-03-28

# 2. 批量分析一段时间
python analyze_turnover_surge.py --start 2025-01-01 --end 2026-03-28

# 3. 调试：列出行业代码格式
python analyze_turnover_surge.py --show-industries
```

## 7. 依赖数据源
脚本依赖于本地 `stock_data.db` 数据库中的以下表：
*   `daily_kline`: 日线 K 线数据（收盘价、换手率等）。
*   `fundamentals`: 基本面数据（总市值等）。
*   `stock_industries`: 行业分类数据。
*   `moving_averages`: 预计算的移动平均线数据。
*   `trade_calendar`: 交易日历。
